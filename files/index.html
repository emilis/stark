Now lets look through the contents of your site directory:

build/Output directory. Stark will create it if it is missing.
index.mpcHome page. Starting point. Required.
config.mpcSite-wide configuration. Optional.
lib/Components directory. Optional.
pages/Pages directory. Optional.
static/Static files directory. Optional.

build/

You will find the compiled pages, scripts, styles and other files here. Directory contents should look like this:

index.htmlHome page created from index.mpc
style.cssCSS style built from requirements of index.mpc
script.jsJavaScript modules build from requirements of index.mpc
static/Files copied over from static/
somepage/ index.htmlPage created from pages/somepage.mpc

index.mpc

This is the main page of the site and the starting point for CSS and JavaScript generation:

  • Stark creates build/index.html from jst/html/markdown part of this page.
  • Stark creates build/script.js and build/style.css from the requirements of this page.

config.mpc

The data from the yaml part of this file will be available as site.config value in jst templates.

Alternatively you can rename the file to config.yaml. (Don't forget to remove the MPC part headers).

pages/

Directory for your page components. pages/yourpage.mpc will be used to generate build/yourpage/index.html.

static/

The contents of this directory are recursively copied to build/static.

lib/

This is the directory for templates, widgets, JS modules, third party libraries. You should probably put source files into subdirectories and not to the lib/ itself, e.g.:

  • lib/jquery/jquery.js
  • lib/yourproject/layout.mpc
  • lib/yourname/widely-reused-library.mpc
